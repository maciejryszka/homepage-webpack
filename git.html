jest git!

System kontroli wersji GIT dostępny jest na tej stronie.
Jeden projekt = jedno repozytorium!
Source Control/GitLens to narzędzia ułatwiające pracę z GITem w Visual Studio Code.
Po zainstalowaniu należy skonfigurować użytkownika wykonując polecenia:

  • git config --global user.name "Imię Nazwisko"
  • git config --global user.email adres_email

Dane użytkownika można sprawdzić z użyciem poleceń:

  • git config --get user.name
  • git config --get user.email

Polecenia GITa:

  • git init Inicjalizacja repozytorium (należy być w katalogu z projektem).
  • git status Sprawdzenie stan repozytorium (śledzone / nieśledzone pliki, ostatni commit, aktualną gałąź).
  • git add . Dodanie plików z aktualnego katalogu oraz podkatalogów jako śledzone.
  • git commit -m "komentarz" Zapis plików. Utworzenie commita ('wersji' projektu).
  • gitk Wyświetlenie historii commitów.

GIT podsumowanie:

  • GIT pozwala na sprawne zarządzanie zmianami w projekcie.
  • Działa lokalnie.
  • Do działania zdalnego potrzebuje serwera (np. GitHub).
  • Commity należy wykonywać po wykonaniu widocznej zmiany w projekcie (lepiej za często niż za rzadko).
  • Podstawowy flow: git add ./git commit -m "komentarz"/git push

github

Serwis służący do przechowywania repozytoriów online to GitHub.
Nowe repozytorium zdalne można utworzyć za pomocą opcji New repository (po wcześniejszym zalogowaniu do serwisu)
Nazwa repozytorium zdalnego powinna być taka sama jak nazwa repozytorium lokalnego.
Po utworzeniu repozytorium zdalnego można albo zainicjalizować nowe repozytorium lokalne albo podpiąć się do już istniejącego (odpowiednie komendy dla obu przypadków wyświetlają się po utworzeniu repozytorium zdalnego).
Kiedy repozytorium zdalne i lokalne są już połączone, aby zaktualizować repozytorium zdalne należy jedynie użyć metody git push po wykonaniu commita.

github pages: statyczny hosting

Serwis 'GitHub Pages' umożliwia publikację strony. Opcja taka znajduje się w ustawieniach repozytorium zdalnego w sekcji 'GitHub Pages'.

gitignore

Plik gitignore zawiara zestaw reguł, pozwalających na zdefiniowanie listy plików/katalogów, które będą znajdowały się w projekcie, lecz nie będą częścią repozytorium. Aby utworzyć taką listę, należy w głównym katalogu repozytorium stworzyć plik o nazwie .gitignore, a następnie umieścić w nim nazwy plików/katalogów, które chcemy wyłączyć ze śledzenia przez Git.

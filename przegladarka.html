jak działa internet i przeglądarka

Zawartość URL (Uniform Resource Locator):

  • protokół
  • host (adres serwera)
  • port (ukryty)
  • root
  • ścieżka dostępu do zasobu
  • parametry (czasami ukryte)

kody odpowiedzi HTTP:

  • 1xx: należy poczekać na odpowiedź.
  • 2xx: OK.
  • 3xx: przekierowanie, należy szukać zasobu gdzie indziej.
  • 4xx: zasób nie został znaleziony.
  • 5xx: błąd serwera.

Protokół obsługujący strony internetowe to HTTP (coraz częściej HTTP/2)
Komunikacja między przeglądarką a serwerem odbywa się na zasadzie request - response.
Adres do zasobu bez '/' z przodu powoduje, że adres ten staje się adresem względnym i przeglądarka szuka pliku w aktualnym katalogu. Nazleży pamiętać o umieszczaniu '/' przed ścieżką do zasobu w przypadku gdy pracujemy z wykorzystaniem serwera.

dom

Document Object Model to 'złożony' w przeglądarce HTML.
JavaScript może modyfikować DOM.
Funkcja 'querySelector' zwraca węzeł (node) z HTMLa, selektor jest zapisywany zgodnie z CSS. Należy korzystać z tej metody i unikać metod: 'getElementById' oraz 'getElementsByClassName'.
Korzystając z BEM należy stworzyć dodatkową klasę z modyfikatorem --js, który ułatwia identyfikację i usuwa potencjalne problemy (np. kiedy edytujemy klasę służącą do stylowania, JavaScript w dalszym ciągu będzie mogła poprawnie pobrać dany element). Jeśli wybierzemy za pomocą funkcji 'querySelector' element, którego nie ma w DOM, zostanie zwrócona wartość null.
Treść elemetnu znajdującego się na stronie można zmieniać z pomocą właściwości 'innerHTML'.

klan

Pojedynczy element drzewa DOM to węzeł/node.
Jeśli element nie posiada zdefiniowanego dla siebie danego stylu, dziedziczy go od najbliższego elementu nadrzędnego, który posiada ten styl. Wyjątkiem jest sytuacja, kiedy dany styl jest predefiniowany dla danego elementu w stylach przeglądarki, np. właściwość 'color' dla znacznika 'a'.
Element z predefiniowanym stylem może odziedziczyć właściwość od elementu nadrzędnego, jeśli dla danej właściwości tego elementu nadamy wartość: 'inherit'.
